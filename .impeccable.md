## Design Context

### Users

General audience using an AI chat service. Users interact with the AI in everyday contexts, looking for quick conversational responses.

### Brand Personality

- Quick: Responsive, snappy interactions with minimal latency perception
- Minimal: Clean, uncluttered interface that focuses attention on the conversation
- Playful: Subtle colorful accents that add personality without overwhelming the minimal aesthetic

Emotional goal: Provide a calm, focused chat experience that still feels welcoming and enjoyable through subtle playful touches.

### Aesthetic Direction

- **Visual style**: Minimalist base with strategic colorful/playful accents
- **Theme**: Supports both light and dark modes, auto-detects system preference
- **Color**: Neutral base palette with one or two accent colors for highlights and interactive elements
- **Typography**: Clear hierarchy with a distinctive but restrained display voice paired with highly readable body text
- **Motion**: Subtle, appropriate animations and transitions that enhance the experience without being distracting

### Design Principles

1. **Content first**: The conversation content is the focus - UI elements should support reading and writing, not compete for attention
2. **Minimal with personality**: Strict minimalism in layout and structure, small playful touches in color and detail to avoid feeling cold or generic
3. **Responsive by default**: Quick interactions with optimistic UI patterns, smooth transitions between states
4. **Both modes**: Full support for both light and dark themes that adapt seamlessly to system preferences
5. **Purposeful motion**: Only animate when it communicates state or improves orientation - no gratuitous movement

### Tailwindcss v4

1. Don't write inline CSS and inline styles.

```jsx
// don't write this
<div  style={{ top: '20%', left: '30%' }} />

// don't write this
<style jsx>

</style>
```

2. Write unique CSS classes using the Tailwind CSS theme API，Like Animation

   ```css
   @import "tailwindcss";
   @theme {
     --animate-wiggle: wiggle 1s ease-in-out infinite;
     @keyframes wiggle {
       0%,
       100% {
         transform: rotate(-3deg);
       }
       50% {
         transform: rotate(3deg);
       }
     }
   }
   ```

3. Write all custom animations into app/animation.css.

### Components

1. Extract complex components into separate components. For reusable components, write them in the `/components` directory, and for non - reusable components, write directly them in `page.tsx`.
